## 2. Верстка головної вебсторінки для поштової розсилки (Вимога 2, 3)

### Метод верстки та аргументація
Для забезпечення максимальної сумісності з усіма поштовими клієнтами (веб-інтерфейсами gmail.com, mail.ukr.net, mail.i.ua та десктопними програмами, як-от Mozilla Thunderbird), для версії поштової розсилки було застосовано **табличну верстку (`<table>`)**. Стилі були максимально **вбудовані (inline)**, оскільки поштові клієнти мають дуже обмежену підтримку сучасного CSS.

### Переваги та Недоліки Видів Верстки для Email

| Вид Верстки | Переваги | Недоліки |
| :--- | :--- | :--- |
| **Таблична (`<table>`)** | **Універсальність та надійність** відображення в більшості поштових клієнтів. | **Громіздкість** коду, складнощі з повною адаптивністю. |
| **Сучасна (Flexbox/Grid)** | **Чистота** коду, легкість створення адаптивного дизайну. | **Низька сумісність:** Більшість поштових клієнтів ігнорують або некоректно відображають ці макети. |

### Підтвердження надсилання

## 3. Адаптивна верстка для смартфону та інтеграція (Вимога 4, 5, 6)

### Реалізація мобільної версії
Для забезпечення комфортного використання на смартфоні інтегрована **адаптивна верстка** за допомогою медіазапиту: `@media screen and (max-width: 600px)`.

### Аргументація змін позиціювання та інтеграція

У десктопній версії контент був розділений на дві секції: `.left_section` (блоки 1, 3, 4) та `.right_section` (блоки 2, 5). Для мобільної версії необхідно було **змінити порядок елементів** на єдиний вертикальний потік, інтеркалюючи блоки з різних секцій.

Застосовано **CSS-Only workaround** з використанням Grid-макету:

| CSS Властивість | Застосування | Результат |
| :--- | :--- | :--- |
| **`.whole_body`** | `display: grid; grid-template-columns: 1fr;` | Переведення основного контейнера у Grid для управління вертикальним потоком. |
| **`.left_section`, `.right_section`** | `display: contents;` | **Ключовий прийом:** Видалення секцій з потоку. Вкладені елементи (`.one`, `.two`, `.three`, і т.д.) стали прямими Grid-елементами контейнера `.whole_body`. |
| **Порядок блоків** | `grid-row: X / span 1;` | Встановлення необхідного вертикального порядку (наприклад, 1, 2, 3, 4, 5) для кожного контентного блоку, незалежно від його початкового батьківського контейнера. |

**Аргументація:** Зміни забезпечили пріоритет вертикального потоку на мобільних пристроях, виводячи навігаційні елементи вище та гарантуючи, що елементи займають 100% ширини, що критично для читабельності.
